# LeetCode 01: Two Sum (两数之和)

## 解题思路
1. **暴力枚举**：两层循环遍历，找 `nums[i] + nums[j] == target`。缺点是时间复杂度 $O(n^2)$，处理大数据量会超时。
2. **哈希表**：遍历一次。每次检查 `target - nums[i]` 是否已在字典里。在就返回，不在就存入字典。时间复杂度降为 $O(n)$。

## 易错点
- 下标是从 0 开始的。
- 题目明确“每种输入只会对应一个答案”，所以不需要处理多解情况。
- 如果数组里有重复数字（如 `[3, 3]`, target 为 6），哈希表的处理顺序很重要。

## 复杂度分析
- 时间：$O(n)$
- 空间：$O(n)$，因为最坏情况下要存入所有元素到字典。
